## Выводы

1. **Позитивные сценарии** не проходят из-за критических багов в работе с `order_entity`:
    - При оплате в `order_entity.payment_id` записывается `transaction_id` вместо `id` (Issues #007, #008)
    - При кредите поле `credit_id` остается `NULL` (Issues #001, #006)
2. **Негативные банковские сценарии** не работают — карта `4444 4444 4444 4442` ошибочно одобряется (Issues #002-005).
3. **Валидация поля "Владелец"** полностью отсутствует — любые невалидные значения принимаются (Issues #029-048).
4. **Поле CVC** пропускает невалидное значение `000` (Issues #057-060).
5. **Сообщения для пустых полей** не соответствуют плану (Issues #013-016, #021-028, #049-052, #060-063).
6. **Карта не из эмулятора** вызывает двойное уведомление (Issues #009-012).

## Рекомендации

1. Исправить связь с `order_entity`:
    - При оплате в `payment_id` записывать `id` из `payment_entity`, а не `transaction_id`
    - При кредите в `credit_id` записывать `id` из `credit_request_entity`
2. Настроить корректную обработку карты со статусом DECLINED.
3. Добавить валидацию для поля "Владелец" (только латиница, имя и фамилия через пробел).
4. Запретить ввод CVC `000` и добавить валидацию.
5. Исправить сообщения для пустых полей.
6. Устранить конфликт уведомлений при использовании карт не из эмулятора.
