# Итоговый отчёт по автоматизации тестирования

## 1. Объект тестирования

Веб-сервис "Путешествие дня", предоставляющий возможность покупки тура с помощью дебетовой карты или в кредит. Приложение взаимодействует с эмулятором банковских сервисов (gate-simulator) и поддерживает работу с СУБД MySQL и PostgreSQL.

**Репозиторий:** [GitHub](https://github.com/MayaPavlova987/Diplom)

## 2. Что было сделано

### 2.1. Настройка окружения
- Разработаны Dockerfile и docker-compose.yml для запуска контейнеров с MySQL, PostgreSQL и gate-simulator.
- Настроен Gradle с необходимыми зависимостями (JUnit 5, Selenide, Allure, DBUtils, JavaFaker, Lombok).

### 2.2. Разработка тестов
- Созданы Page Objects для страниц "Оплата" и "Кредит".
- Разработан DataGenerator для генерации тестовых данных.
- Разработан DbUtils для работы с базами данных.
- Написаны автотесты для 35 тестовых сценариев (всего 66 тестов с учётом двух БД и двух форм).
- **Важно:** Сценарий TC-8 (17 цифр в номере карты) был модифицирован из-за технического ограничения приложения:
  поле ввода не позволяет ввести 17-ю цифру. Вместо проверки сообщения об ошибке
  тест проверяет ограничение максимальной длины поля. Это решение признано корректным, так как иного
  способа проверить валидацию в данной ситуации нет.

### 2.3. Прогон тестов
- Тесты выполнены на двух СУБД: MySQL и PostgreSQL.
- Сформированы отчёты Allure и Gradle.
- Зафиксированы результаты и оформлены **67 баг-репортов** в GitHub Issues, включая:
    - [#001](https://github.com/MayaPavlova987/Diplom/issues/1) — кредит MySQL
    - [#006](https://github.com/MayaPavlova987/Diplom/issues/6) — кредит PostgreSQL
    - [#007](https://github.com/MayaPavlova987/Diplom/issues/7) — оплата MySQL
    - [#008](https://github.com/MayaPavlova987/Diplom/issues/8) — оплата PostgreSQL

## 5. Оценка готовности продукта

**Продукт не может быть выпущен в промышленную эксплуатацию** из-за наличия критических дефектов, влияющих на корректность обработки платежей и сохранения данных.

### Требуется доработка:
- Исправление связи с `order_entity`:
    - При оплате в `payment_id` записывать `id` из `payment_entity`, а не `transaction_id`
    - При кредите в `credit_id` записывать `id` из `credit_request_entity`
- Корректная обработка отклонённых карт
- Добавление валидации поля "Владелец"
- Исправление сообщений для пустых полей
- Запрет CVC `000`
- Устранение конфликта уведомлений
